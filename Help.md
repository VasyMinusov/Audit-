# Помощь по поиску уязвимостей

Этот файл поможет тебе понять, где искать проблемы. Здесь нет готовых решений — только направления для поиска.

## Веб-уязвимости

### SQL-инъекции
Ищи места, где пользовательские данные попадают в запросы к базе. Проверяй формы ввода, параметры URL, заголовки запросов. Смотри, как приложение обрабатывает специальные символы и кавычки.

### XSS (межсайтовый скриптинг)
Проверяй все поля, куда пользователь может ввести текст. Смотри, как этот текст отображается на странице. Обращай внимание на места, где данные выводятся без обработки.

### SSRF (подделка запросов)
Ищи функционал, который делает запросы к внешним ресурсам. Проверяй параметры с URL, webhook-и, загрузку файлов по ссылке. Смотри, есть ли ограничения на адреса.

### RCE (выполнение кода)
Ищи места, где данные пользователя могут попасть в системные вызовы. Проверяй загрузку файлов, выполнение команд, работу с оболочкой. Смотри на обработку расширений файлов.

## Уязвимости конфигурации

### Небезопасные заголовки
Проверяй HTTP-заголовки ответов сервера. Смотри на заголовки безопасности, информацию о сервере, политики CORS.

### Проблемы аутентификации
Изучай механизмы входа и сессий. Проверяй хранение токенов, сроки действия, механизмы сброса пароля. Смотри на разграничение прав доступа.

### Утечки информации
Ищи места, где приложение раскрывает лишние данные. Проверяй сообщения об ошибках, служебные эндпоинты, метаданные файлов.

## Уязвимости бизнес-логики

### Обход ограничений
Проверяй сценарии использования функционала. Смотри, можно ли нарушить предполагаемый порядок действий. Обращай внимание на проверки условий.

### Проблемы авторизации
Тестируй доступ к ресурсам разных пользователей. Проверяй, можно ли получить доступ к чужим данным через изменение идентификаторов.

## Где искать

1. Исходный код приложения — читай файлы в папке backend/app
2. Конфигурационные файлы — docker-compose.yml, nginx.conf, файлы окружения
3. Зависимости — requirements.txt, package.json
4. Логи и сообщения об ошибках
5. Сетевые запросы через DevTools браузера

## Инструменты для поиска

- Встроенный поиск по коду в редакторе
- Командная строка с grep или аналогами
- DevTools браузера для анализа запросов
- Docker-логи для просмотра работы приложения
