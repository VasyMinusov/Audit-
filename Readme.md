# Подготовка к занятию по аудиту безопасности

## Почему важно скачать всё заранее

Занятие пройдёт в условиях локальной сети без доступа к интернету. У вас не будет возможности докачать инструменты или образы в процессе работы. Все необходимые файлы должны быть на вашем компьютере до начала занятия. Проверьте наличие каждого инструмента из списка ниже.

## Что нужно установить

### 1. WSL и Ubuntu (рекомендуемый вариант)

Откройте PowerShell от имени администратора и выполните команду:

```
wsl --install
```

Перезагрузите компьютер. Ubuntu установится автоматически после перезагрузки.

Если нужна конкретная версия Ubuntu:

```
wsl --install -d Ubuntu
```

### 2. Инструменты для аудита

После установки Ubuntu добавьте репозиторий Kali Linux и установите необходимые инструменты:

```bash
# Обновите пакеты
sudo apt update && sudo apt upgrade -y

# Добавьте репозиторий Kali
echo "deb http://http.kali.org/kali kali-rolling main non-free contrib" | sudo tee /etc/apt/sources.list.d/kali.list
wget -q -O - https://archive.kali.org/archive-key.asc | sudo apt-key add -

# Обновите список пакетов
sudo apt update

# Установите основные инструменты
sudo apt install -y nmap sqlmap nikto burpsuite curl wget git python3 python3-pip
```

#### Зачем нужен каждый инструмент

- **nmap** — сканирует сеть, находит открытые порты и запущенные сервисы. Вы увидите, какие службы работают на целевой машине.
- **sqlmap** — автоматизирует поиск SQL-инъекций. Инструмент проверяет параметры запросов и находит уязвимые места.
- **nikto** — сканирует веб-сервер на известные уязвимости. Показывает проблемы конфигурации и опасные файлы.
- **burpsuite** — перехватывает и анализирует HTTP-запросы. Вы сможете модифицировать запросы и проверять реакцию сервера.
- **curl** — отправляет HTTP-запросы из командной строки. Удобно для быстрой проверки эндпоинтов.
- **wget** — скачивает файлы по HTTP и FTP. Пригодится для загрузки данных с целевого сервера.
- **git** — работает с репозиториями. Через git вы получите доступ к скриптам и материалам занятия.
- **python3 и pip** — запускают скрипты для аудита. Многие инструменты написаны на Python и требуют интерпретатора.

### 3. Браузер с DevTools

Подойдёт любой современный браузер:

- Google Chrome
- Mozilla Firefox
- Microsoft Edge

Научитесь открывать инструменты разработчика по клавише F12. Вы будете смотреть сетевые запросы, куки, заголовки и ответы сервера.

## Альтернативный вариант: Kali Linux на виртуальной машине

### Важное замечание

Сайт kali.org заблокирован в России. Используйте VPN для скачивания образа.

### 1. Программное обеспечение для виртуализации

Установите одно из следующих решений:

- VirtualBox: https://www.virtualbox.org/
- VMware Workstation Player: https://www.vmware.com/products/workstation-player.html

### 2. Образ Kali Linux

Скачайте готовый образ с официального сайта:

https://www.kali.org/get-kali/#kali-virtual-machines

Выберите образ для вашей платформы виртуализации.

### 3. Запуск виртуальной машины

Импортируйте образ в программу виртуализации и запустите машину.

Стандартные учётные данные:

- Логин: kali
- Пароль: kali

### 4. Обновление системы

После первого запуска обновите систему:

```bash
sudo apt update && sudo apt upgrade -y
```

## Проверка перед занятием

К началу занятия у вас должны работать следующие компоненты:

- WSL с Ubuntu или Kali Linux на виртуальной машине
- Python 3 и pip
- Инструменты: nmap, sqlmap, nikto, curl, wget, git
- Браузер с работающими DevTools

Напишите в чат заранее, если что-то не работает.
